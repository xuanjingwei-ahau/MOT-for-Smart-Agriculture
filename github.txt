第一阶段：首次设置（将本地文件夹与新建的GitHub仓库关联并上传）
请在你的本地新文件夹中，按顺序执行以下命令：

步骤	命令	作用与说明
1. 进入文件夹	cd /你的/文件夹/完整/路径	在终端中导航到你的本地项目文件夹。
2. 初始化仓库	git init	将该文件夹初始化为一个Git本地仓库，创建 .git 子目录。
3. 添加所有文件	git add .	将文件夹内所有文件添加到暂存区。如果只想添加特定类型文件，可用 git add *.py 等。
4. 首次提交	git commit -m "初始提交"	创建第一个本地提交记录。
5. 关联远程仓库	git remote add origin https://github.com/你的用户名/你的新仓库名.git	关键步骤：将本地仓库与你在GitHub上新建的空白仓库建立连接。
6. 首次推送	git push -u origin main	将本地 main 分支的所有内容推送到远程仓库，并建立追踪关系。


首次推送常见错误：如果远程仓库创建时包含 README.md、.gitignore 等初始化文件，会导致历史冲突。此时使用 git push -u origin main --force（强制推送）可覆盖，但需谨慎。


第二阶段：后续更新（修改文件后同步到GitHub）
完成首次设置后，后续的任何文件修改、添加、删除，都遵循以下固定循环：

步骤	命令	说明
1. 添加变更	git add . 或 git add 文件名	将工作区的变更放入暂存区。
2. 提交变更	git commit -m "描述更改"	将暂存区的变更打包为一个新的本地提交。
3. 推送更新	git push	由于首次推送已设置上游，此处直接推送即可。
